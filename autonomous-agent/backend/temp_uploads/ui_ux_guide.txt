UI/UX Design Guidelines - Checkout Page

Color Scheme & Feedback

Primary Action Button: The "Pay Now" button must be styled with a Green background color (e.g., #008000 or similiar) to indicate a positive action.

Add to Cart Buttons: Should be distinct from the Pay Now button, preferably Blue.

Success Messages: Upon successful payment, a message saying "Payment Successful!" must appear in a light green box.

Form Validation Visualization

Error Messages: All validation errors (e.g., "Name is required") must be displayed in Red text.

Placement: Error messages should appear immediately below the invalid input field.

Timing: Errors should appear when the user attempts to submit the form with invalid data.

Layout & Responsiveness

The layout should be contained within a centered box with a maximum width of 800px.

Inputs should span the full width of their container for better readability on mobile devices.

The "Total Price" should be prominently displayed in bold text, aligned to the right.

Default States

"Standard Shipping" radio button must be pre-selected.

"Credit Card" payment radio button must be pre-selected.

Error messages must be hidden by default.
